## 三、犯过的错误

### 错误 1：WorkflowNodeConfig outputs 类型不匹配
- **现象**：`process_from_dicts` 构造 Pydantic 对象时报错 `outputs: Input should be a valid dictionary, input_value=[]`
- **原因**：`benchmark_adapter.py` 中 node 的 `outputs` 设为 `[]`（list），但 Pydantic 模型期望 `Dict` 或 `None`
- **修复**：改为 `"outputs": None`

### 错误 2：All-at-Once 输出解析失败
- **现象**：gpt-5.4-nano 输出格式带 markdown 加粗 `**`，正则 `Agent Name:\s*([\w_]+)` 匹配不到
- **原因**：ISSUES.md 问题五的变体，LLM 不严格遵循格式
- **修复**：正则改为兼容 `\**` 前缀，同时更新 evaluate.py 的解析正则

### 错误 3：step_mae 输出 NaN
- **现象**：`test_e2.json` 中 `"step_mae": NaN` 导致 JSON 不合法
- **原因**：当没有 Agent 预测正确时 `step_mae` 返回 `float('nan')`
- **修复**：改为返回 `None`（JSON 中显示为 `null`）

### 错误 4：E2 意图识别 prompt 没改模型
- **现象**：用户要求确认所有模型都改成 gpt-5.4-nano
- **遗漏**：`intent_recognition.hprompt` 还是 `gemini-2.5-pro`
- **修复**：改为 `gpt-5.4-nano`

### 错误 5：误跑全量数据
- **现象**：测试原论文方法时调用 `all_at_once(..., test_mode=False)`，导致 126 条全部开始跑
- **原因**：手动调用函数时忘了传 `test_mode=True`
- **修复**：后续改用 `/tmp/benchmark_test/` 单文件目录测试

### 错误 6：可视化图表问题
- **分层对比图**：多方法柱子重叠 → 改为 x 偏移并排
- **效率对比图**：LLM Calls(1) 和 Latency(6000) 同轴看不到 → 拆成双子图
- **对比表 nan**：Step MAE 为 None 时显示 nan → 改为显示 `-`
